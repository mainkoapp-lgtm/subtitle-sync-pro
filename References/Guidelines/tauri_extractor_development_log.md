# Tauri 자막 추출기 개발 및 트러블슈팅 기록

## 2026-04-22: 자막 추출 로직 고도화 및 네트워크 드라이브 이슈 해결

- **문제점**:
    1. **저장 실패**: 무확장자 임시 파일(.tmp) 사용 시 FFmpeg가 출력 형식을 인식하지 못해 `Invalid argument` 에러 발생.
    2. **진행률(%) 미표시**: 네트워크 드라이브에서 `ffprobe` 호출 시 지연/프리징 발생 및 `stderr` 파싱의 불안정성으로 인해 진행률이 0%에 고정됨.
    3. **중단 시 응답 없음 (ANR)**: 동기식(Sync) I/O 및 프로세스 대기 로직으로 인해 네트워크 지연 시 UI 스레드와 백엔드 통신이 차단됨. 또한 `taskkill /T` 옵션이 상위 프로세스에 영향을 주어 앱이 강제 종료됨.

- **수정 과정 및 핵심 코드**:
    1. **비동기화 (Async/Tokio)**: 모든 Tauri 커맨드를 `async`로 전환하고, `tokio::process::Command` 및 비동기 I/O(`BufReader`, `lines()`)를 사용하여 프리징 방지.
    2. **정밀 파싱 (-progress pipe:1)**: 사람이 읽는 로그 대신 기계용 진행률 옵션을 도입하여 `out_time_us`를 파싱. `stderr`에서는 별도 비동기 태스크로 `Duration`을 추출하여 취합.
    3. **안전한 종료**: `taskkill`에서 트리 종료(`/T`) 옵션을 제거하고 개별 PID만 종료하여 시스템 안정성 확보.
    4. **무결성 검사**: `fs::copy` 등 파일 작업의 에러를 무시하지 않고 프런트엔드까지 전달하도록 강화.
    5. **이름 중복 회피**: 동일 파일명 존재 시 `(2)`, `(3)` 접미사를 자동으로 추가하여 데이터 유실 방지.

### 2026-04-22: 배너 클릭 이동 이슈 및 샌드박스 정책 적용
- **문제점**: 서버 기반 HTML 배너 노출에는 성공했으나, 배너 클릭 시 외부 브라우저가 열리지 않는 현상 발생. 
- **원인 분석**:
  1. **Tauri iframe 보안**: WebView2 환경에서 iframe 내부의 `window.open` 및 `target="_blank"`는 부모 윈도우의 허용 없이는 자동으로 차단됨.
  2. **플러그인 격리**: iframe 내부는 원격 URL(Firebase)에서 로드되므로 Tauri API(`@tauri-apps/plugin-opener`)를 직접 호출할 수 없음.
- **수정 과정 및 핵심 코드**:
  1. **샌드박스 속성 적용**: `App.tsx`의 iframe 태그에 `sandbox="allow-scripts allow-popups allow-popups-to-escape-sandbox allow-same-origin"`을 추가하여 팝업 차단 정책을 완화함.
  2. **표준 a 태그 전환**: `onclick` 방식에서 `<a>` 태그 기반의 표준 링크 구조로 수정하여 WebView의 링크 가로채기 호환성을 높임.
  3. **서버 동기화**: 수정된 HTML 형식을 Firebase Hosting에 최종 배포 완료.
- **결과**: [성공] (배너 노출 및 서버 연동 완료 / 클릭 이슈는 기술적 제한 사항으로 기록)

### 2026-04-22: iframe 샌드박스 한계 극복을 위한 투명 오버레이 적용
- **문제점**: Tauri 앱에서 원격 `iframe`을 사용할 경우, 도메인이 다르기 때문에(Cross-Origin) `postMessage`를 이용한 클릭 이벤트 전달이 보안 정책에 의해 차단되거나 신뢰성 있게 작동하지 않음. 이로 인해 `openUrl`을 도입했음에도 브라우저가 열리지 않는 현상 지속.
- **해결 방법 및 핵심 코드**:
  1. **Tauri Opener 필수 사용**: Tauri 2.0에서는 `window.open`이 시스템 브라우저를 열지 못하므로 `@tauri-apps/plugin-opener`의 `openUrl`을 사용해야 함.
  2. **투명 오버레이 기법**: `iframe` 내부의 클릭 이벤트에 의존하지 않고, 부모(React) 레이어에서 `iframe` 위에 `z-index`가 높은 투명한 `div`를 덮어씌움.
  3. **이벤트 가로채기**: 사용자가 배너를 클릭할 때 `iframe` 대신 오버레이가 클릭을 수신하게 하여, 부모 컨텍스트에서 직접 `openUrl`을 호출함으로써 보안 제약을 완전히 우회함.
- **결과**: [성공] (2026-04-22 배너 클릭 및 외부 브라우저 연동 성공 확인)
