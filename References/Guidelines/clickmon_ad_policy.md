# 클릭몬 (ClickMon) 광고 운영 정책 및 가이드 (2026-04-22 수집)

## 1. 개요
본 문서는 자막 싱크 서비스(Subtitle Sync Pro) 내 클릭몬 광고 연동을 위한 규격 및 운영 정책을 정의합니다.

## 2. 수집된 광고 상품 종류
| 광고 상품 | 유형 | 특징 | 비고 |
| :--- | :--- | :--- | :--- |
| **MonAD (Web)** | 배너 | 300x250, 250x250 등 다양한 사이즈 제공 | 매체(도메인) 승인 후 코드 발급 |
| **MonAD (Mobile)** | 배너 | 모바일 기기에 최적화된 배너 | User-Agent 감지 후 동적 로딩 필요 |
| **HIT View** | 위젯/박스 | 블로그나 게시판 형태에 적합한 박스형 광고 | |
| **제휴 마케팅** | CPA/CPS | 클릭/구매 발생 시 수익 창출 | 배너형 대비 높은 전환 가치 |
| **전면 광고** | 레이어 | 페이지 진입/종료 시 전체 화면 덮기 | 사용자 경험(UX)에 큰 영향 |

## 3. 코드 연동 방식 (표준 규격)
클릭몬 광고는 기본적으로 **아이프레임(Iframe)** 혹은 **자바스크립트(JS)** 형태로 발급됩니다.
승인이 완료되면 시스템 내에서 아래와 같은 형태의 템플릿 코드를 발급받아 적용합니다.

```html
<!-- 예시 형태 (React 적용 시 로직 변환 필요) -->
<script type="text/javascript">
    // 승인 후 발급되는 실제 식별자로 교체
</script>
<script type="text/javascript" src="https://www.clickmon.co.kr/..."></script>
```

## 4. 운영 주의 사항
*   **승인 절차**: 등록된 도메인(`subtitle.mainko.net`)이 클릭몬 관리자의 최종 승인을 받아야만 실 서버에 광고가 노출됩니다.
*   **무효 클릭 방지**: 짧은 시간 내 동일 IP에서의 반복 클릭은 무효 처리될 가능성이 높으므로, 테스트 시 주의해야 합니다.
*   **로테이션 전략**: 동일 사용자에게 반복적인 노출 시, Monetag 등 다른 광고 네트워크와 확률 기반으로 교차 노출하여 수익 누락을 최소화해야 합니다.
