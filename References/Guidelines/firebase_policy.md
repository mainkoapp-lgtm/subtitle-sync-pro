# 파이어베이스(Firebase) 호스팅 및 요금 정책 요약

## 1. Spark (무료) 요금제 주요 할당량
*   **저장 공간(Storage)**: 프로젝트당 **10 GB**
*   **데이터 전송량(Network Transfer)**: 프로젝트당 **일일 360 MB** (KST 기준 매일 초기화)
*   **커스텀 도메인 및 SSL**: 무료 지원

## 2. 할당량 적용 범위
*   **프로젝트 단위 적용**: 무료 할당량은 구글 계정 전체가 아니라 **개별 프로젝트별**로 각각 부여됨.
*   **독립적 운영**: 프로젝트 A의 일일 전송량을 모두 소진해도 프로젝트 B의 서비스에는 영향을 주지 않음.
*   **멀티 사이트 공유**: 하나의 프로젝트 내에서 여러 개의 앱(Web, Android, iOS)을 운영할 경우, 해당 앱들이 프로젝트의 일일 할당량을 공유함.

## 3. 계정당 프로젝트 한도
*   **기본 한도**: 한 구글 계정당 생성 가능한 무료 프로젝트 수는 일반적으로 **5~10개** 내외임.
*   **한도 증설**: 프로젝트 생성 한도에 도달할 경우 Google Cloud 지원팀에 요청하여 증설이 가능함.

## 4. 권장 운영 전략
*   **서비스 분리**: `FlowStateTimer`, `Subtitle Extractor` 등 서비스 성격이 다른 앱은 각각 별도의 Firebase 프로젝트를 생성하여 운영하는 것이 할당량 확보 및 관리 측면에서 유리함.
*   **대용량 파일 배포**: 설치용 ZIP 파일 등 대용량 데이터는 Firebase Hosting의 일일 전송량(360MB)을 빠르게 소진할 수 있으므로, 외부 저장소(Dropbox 공유 링크 등)를 활용하는 것이 효율적임.
